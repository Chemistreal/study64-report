## 진단 구조

6개 축 × 2극 = 12극, 각 극당 8문항 = 96문항.

| 축 | 두 극 |
|---|---|
| 이해 방식 | 심층이해 / 반복숙달 |
| 실행 방식 | 계획실행 / 상황반응 |
| 학습 구조 | 자기주도 / 외부구조 |
| 막힘 처리 | 질문·확인 / 자기해결 |
| 오류 처리 | 완성추구 / 시행착오 |
| 지속 동기 | 미래목표 / 즉시피드백 |

극 점수 = (해당 8문항 응답 평균) ÷ 5 × 100.

## 사용 방법

### 학생 검사
1. `index.html`을 브라우저로 연다
2. 학생이 96문항에 1~5로 응답 → 리포트 자동 생성
3. 리포트: 12극 점수, 트랙 적합도(영재고/자사고/과학고/메디컬), 학습 처방

### 종이 검사 + 관리자 입력 (반 단위)
1. `PRISM_설문지_OMR.pdf`를 인쇄해 학생에게 배부
2. 학생이 OMR에 표기
3. `index.html?admin=1`로 관리자 패널 진입 (또는 우하단 "＋ OMR 수기입력" 버튼)
4. OMR 보고 키보드로 1~5 연타 입력 (자동 다음 칸 이동)
5. **반 연속 입력**: 이름 → 96자 입력 → `Ctrl+S`(저장하고 다음) → 반복
   - `Ctrl+Enter`: 리포트 생성 / `Esc`: 닫기

## 관리자 도구 (A단계 + B단계 + 품질 개선)

### A단계 · 응답 품질 계측
- **응답 품질 점검**: 직진 응답·극단·중앙 편향·대립문항 모순을 탐지해 신뢰도 0~100점 산출
- **리포트 자동 게이트**: 응답이 의심스러우면 적합도 수치에 "참고용" 표시 자동 부착
- **품질 상세 패널**: 12개 대립쌍 모순 내역, 응답 분포, 직진 구간 시각화
- **검사 기록 저장**: 학생별 자동 저장(이름·답안·12극·품질). 목록·검색·CSV 내보내기
- **백업/복원**: 전체 기록을 JSON 파일로 내보내고 다른 기기에서 불러오기(중복 자동 건너뜀)
- **재검사 비교**: 같은 학생 2회 검사의 12극 변화·신뢰도 추이
- **교사 라벨 수집**: 실제 트랙·관찰 성향·체감 정확도·실제 결과 입력(타당도 검증용 정답 데이터)

### B단계 · 성장 추적
- **다회차 추세선**: 3회+ 검사를 12극 꺾은선으로(색맹 대응: 대시 패턴 + 마커 모양)
- **성장 분석**: 각 극을 꾸준히 오른/내린/출렁인/안정된으로 자동 분류
- **반 단위 집계**: 여러 학생의 12극 평균·분포, 집단 강점·약점·편차 극
- **반 대비 개인 위치**: 한 학생을 반 평균 대비 상대 위치(백분위)
- **극 간 상관**: 어떤 학습 성향끼리 같이 움직이는지 상관 행렬(표본 크기 경고 포함)
- **추세 HTML 내보내기**: 상담·학부모용 독립 문서

### 운영·품질 개선
- **채점 감사**: 12극 점수를 문항 단위로 역추적(입력 실수 검산)
- **입력 무결성**: 같은 이름 중복·미입력·빈 이름 경고
- **저장 용량 관리**: localStorage 사용량 미터, 정리 도구, 저장 실패 경고
- **라벨 CSV(검증용)**: 검사 결과 + 교사 라벨 결합 CSV(엑셀/R 분석용)
- **데이터 스키마 버저닝**: 미래 구조 변경 시 자동 마이그레이션 + 손상 데이터 자동 복구
- **색맹 안전**: 흑백으로 변환해도 구분되는 중복 부호화
- **모바일 대응**: 360px에서 기록 목록이 카드형으로 자동 전환

## ⚠️ 중요 · 아직 남은 과제 (코드로 해결 불가)

PRISM의 모든 운영·품질 기능은 완성됐지만, **타당도 검증은 아직 안 됐다.** 12극·트랙 적합도·기준선은 임상 직관으로 설계한 가설이며, 실제 학생 데이터로 검증된 적이 없다.

검증에 필요한 것:
1. 실제 학생 30~50명 검사 (도구 준비 완료)
2. 각 학생에 교사 라벨 입력 (실제 트랙·성향·결과는 "라벨" 버튼)
3. **라벨 CSV(검증용)** 내보내기 → 엑셀/R로 분석
4. "검사가 영재고로 본 학생이 실제로 영재고 지망이었나" 같은 준거 타당도 확인

`기준선_검토용.md`의 트랙 기준선도 AI 추정값이므로 실제 합격생 데이터로 교체해야 한다.

**즉, 다음 단계는 코드가 아니라 데이터다.** 이 도구는 그 데이터를 받아 검증할 준비가 모두 끝났다.
